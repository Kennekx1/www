# Skill: Localization Guard

Скилл для обеспечения безупречной мультиязычности.

## Основные проверки
1. **Key Match**: Проверка, что каждый ключ в `ru.json` имеет свой эквивалент в `kk.json`.
2. **Empty Values**: Поиск пустых строк или технических заглушек.
3. **Context Check**: Проверка того, что перевод соответствует "Luxury" контексту (избегать формализма, использовать вдохновляющий язык).

## Структура файлов
- `src/i18n/ru.json` — мастер-файл.
- `src/i18n/kk.json` — перевод.

## Правила перевода на казахский (KK)
- Сохранять эмоциональную окраску оригинала.
- Использовать современные термины (например, `Мәзір` вместо `Меню`, если это уместно для дизайна).
- Проверять длины строк: казахские слова часто длиннее русских на 20-30%, что может сломать кнопки.

## Работа с багами
Если на сайте виден текст вида `COMMON.SELECT_VOLUME`:
1. Найти ключ в коде.
2. Проверить наличие в обоих JSON файлах.
3. Проверить правильность вызова `t('key')`.

## Автоматизация
- Использовать `grep` для поиска нелокализованных русских строк в `.tsx` файлах.
- Команда: `grep -rP '[а-яА-ЯёЁ]' src --include='*.tsx' --exclude='*.test.tsx'`
